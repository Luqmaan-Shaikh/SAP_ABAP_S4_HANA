# ABAP -

-> ABAP Source code consists of either a Statement or a comment

-> Statements - Collection of Operators , Operands , Variables and Keywords. EX :- c = a + b  
-> Comments :- Non executable statements . To comment we use ASTERIK (*----*). EX :- * This is a Comment * 


-> Operators :- 
	- Mathematical (Addition, Subtraction, Multiplication, Exponential, Division, Mod)
	- Comparative (less then , less than equal to , greater than , greater than equal to , equal to , not equal to)
	- Logical (AND, OR, NOT)

-> Operands :- variables which we need to perform particular operations

-> Variables :- Name given to Memory Location

-> Keywords :- Identify type of Statements (C2 B3 E O)
	- Calling
	- Controlling
	- Declaration :- used to declare variable in ABAP (Data, Parameter, Tables, Types, etc)
	- Definition 
	- Database
	- Events
	- Operations


-> Data Types :- 

1. Numeric Data Types 
	- Integer I
	- Float F
	- Packed Decimal P

2. Character Data Types 
	- Char C (Alpha - Numeric) (Integer + Characters)
	- Numeric Char N
	- Date D
	- Time T


- Int, Float, Date, Time -> Fixed Length Data Type
- Rest are -> Variable Length Data Types


NOTE -> Everything you write in ABAP MUST END with a FULL STOP (.)


# Syntax of -

1. Integer (I) 

-> Data <variable_name> type I.
Ex. data A type I.
Initial / Default value of Integers is '0'


2. Float (F) 

-> Data <variable_name> type F.
Ex. data A type F.
Initial / Default value of Float is '.00'


3. Packed Decimal (P) 

-> Data <variable_name> (<length>) type P decimals <no>.
Ex. data A(10) type P decimal 3.
Initial / Default value of Packed Decimal is '.000' (number of decimals)



4. Char (C) (Alpha Numeric)

-> Data <variable_name> (<length>) type C.
Ex. data A(5) type C.
Initial / Default value of Char is 'Space / Empty'


5. Date (D)

-> Data <variable_name> type D.
Ex. data A type D.
Initial / Default Format of Date is 'YYYYMMDD' -> (20140425)


6. Numeric Char (N) (Ex. Contact Numbers , Bank Account Numbers)

-> Data <variable_name> (<length>) type N.
Ex. data A(10) type N.
Initial / Default Value of Numeric Char is '000' (Based on Length)


7. Time (T)

-> Data <variable_name> type T.
Ex. data A type T.
Initial / Default value of Time is '000000'. Format -> 'HHMMSS'

------>>>>>> DATA DICTIONARY :-

- Central source of DBMS 
- Main functionality of DDIC is to create or alter tables

-> 2 ways to create DB Table :-

1. Direct method / pre defined / built in [We Don't use this]
2. Data Element Method (TD Approach , BU Approach)



-> Technical Requirements to create Table :- 
	- Name of table starts with y or z 
	- Provide list of fields, data types and lengths
	- Provide the delivery class
	- Provide the Technical settings



-> Delivery Class :-
- It defines the owner of the tables as well as it controls the transport of the data from one table to another table 
- Technical settings is nothing but the combination of DATA CLASS and SIZE OF THE CATEGORY


-> Data Class :- 
- Data Class defines the physical area of the DB in where our table is logically stored

- Some of the important Data Classes are :-

1. APPL 0 - Master Data = Access Frequently But Updates Rarely (Customer master data, employee master data)
2. APPL 1 - Transaction Data = Access Rarely and Updates Rarely (Sales order , Purchase Order Data) . Technical Data is Dependent Data.
3. APPL 2 - Organization Data = Access Frequently But Updates Rarely (Company Data , Branches Data) . Organizational data is created at the time of system configures.



-> Size Category 
- It determines the space require for the table. 
- Size category : 0
- Each table must have at least one field as primary field that should be the character data type that should be character data type that should in th first field in the table

# Types of Table :-

1. Transparent Tables [We use this]

-> One - to - One R/s
-> If you create one transparent table in DDIC, then it'll store like only one data table in the data base
 
2. Cluster Tables

-> Many - to - One R/s
-> If you create many clustered tables in DDIC then they will form a table cluster and store in the data base
-> Buffering isn't possible for clustered table. For this reason only fetching the data from clustered table takes more time
-> Clustered table is suitable when you fetch the fewer amounts of data from more fields

3. Pool Tables

-> Many - to - One R/s
-> If you create many pooled tables in DDIC, then they will form form like a table pool and stored in the data base
-> Pooled tables are suitable when we fetch the large amount of data from fewer fields

NOTE:- Joins aren't possible for clustered and Pooled Tables

# Internal Tables ******************** :-

1. Internal Tables is the collection of Records.
2. IT are Temporary Tables. Means Data in IT wont be saved anywhere in SAP.
3. IT are Dynamic Memory Location. Means we don't have to provide size of IT.
4. Scope of IT is up to that program.
5. Placing Data to IT as well as getting data from IT is always record by record.


#           Database Table                  VS             Internal Table

- DBT are Permanent Storage Location        VS       IT are Temporary Storage Location
- We can Access DBT from anywhere in SAP    VS      We can Access IT within the Program only
- We must provide Size of DBT               VS      IT are Dynamic, Need not provide the size


# Syntax of declaring IT :-
-> 
Data <internal table> LIKE table of <work area>.

# Ex:- 
-> 
Data IT like table of WA.

# Syntax of accessing fields from Work Area
-> 
<work area name> - <field name>.

# Ex :-
-> 
WA - Eid.

NOTE -> APPEND is the Keyword to Transfer the data from work area to IT


# Ex :- 
->
Data: Begin of emp,
      eid(10) type c,  
      ename(25) type c,
      ecity(30) type c,
      ecntno(10) type numc,
      End of emp.
Data IT like table of WA.
WA-Eid = '1'.
WA-Ename = 'Ali'.
WA-Ecity = 'Hyd'.
WA-Ecntno = '0000000000'.
APPEND WA to IT.
WA-Eid = '2'.
WA-Ename = 'Luqmaan'.
WA-Ecity = 'Mumbai'.
WA-Ecntno = '1111111111'.
APPEND WA to IT.
Loop at IT into WA.
Write:/ WA -Eid, WA - Ename, WA - Ecity , WA - Ecntno.
Endloop.

-> Accepts Duplicate Records
-> Here all fields are Non-Unique
-> Passing Data from work area to IT is always through 'Append' Keyword
-> Searching of a Record is Linear Search

# Syntax :-
->
Data <IT> like standard table of <work area>.


# Sorted IT :-

-> May or may not accept Duplicate Records
-> Here we must specify at least one field as Unique / Non-Unique
-> Passing Data from work area to IT is always through 'Insert' Keyword
-> Searching of a Record is Binary Search

# Syntax :-
->
Data <IT> like sorted table of <work area> with unique / non-unique key <filed>.

#Ex :-
->
Data IT like sorted table of WA with unique key Eid.


# Hashed IT :-

-> Wont accept Duplicate Records
-> Here we must specify at least one field as Unique
-> Passing Data from work area to IT is always through 'Collect' Keyword
-> Searching of a Record is Hash Algorithm

# Syntax :-
->
Data <IT> like hash table of <work area> with unique key Eid.


# Kind is the keyword which returns type of IT 
- Standard (T) , Sorted (S) , Hash (H)

# Lines is the keyword which returns the no of records in IT

# Syntax :-
->
Describe table <internal name> KIND <variable1> LINES <variable2>.

# Ex :-
->
Describe table IT_KNA1 KIND v1 LINES v2.

NOTE -> By Default variable is the character data type and length is 1
NOTE -> By Default IT is Standard IT (T)


# Types of Declaring IT :- 

1. Declaring IT with some of the fields from any one of the dbt :- 

# Ex :-
->
Data: begin of wa_t001,
bukrs type t001-bukrs,
ort01 type t001-ort01,
end of wa_t001.
Data IT like table wa_t001.

2. Declaring IT with all fields from any one of the dbt :-

# Syntax 
-> 
Data begin of <work area>.
Include Structure <DBT>.
Data end of <work area>.
Data <I T> like table of <work area>.

# Ex :-
->
Data begin of wa_t001.
Include Structure t001.
Data end of wa_t001.
Data it_t001 like table of wa_t001.

3. Declaring IT table by Referring DBT (Rarely we use this)

# Syntax :-
->
Data <w a> like <dbt>.
Data <I T> like table of <w a / dbt>

# Ex :-
->
Data wa like t001.
Data it like table of wa.

OR

->
Data IT like table of t001.
Data wa like line of IT.


NOTE :-

# Type is used to Refer :-
	- Data Types
	- Data Elements
	- Fields in the table
	- Types

# Like is used to Refer :-
	- Variables
	- Fields in the table
